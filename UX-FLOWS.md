# Batman's Utility Belt — UX Flows

## Flow 1 — Manager starts the day

1. Manager opens dashboard
2. Sees team rollup cards with pace labels
3. Identifies who is “Needs pressure”
4. Clicks into that team member
5. Reviews open tasks, blockers, and must-win standards
6. Uses coaching prompts to intervene

Outcome:
The manager quickly knows where to focus attention.

## Flow 2 — Team member plans the day

1. User opens their board
2. Sees quota and current task list
3. Adds missing tasks
4. Reviews must-win standards
5. Starts completing actions

Outcome:
The day becomes explicit and countable.

## Flow 3 — Task completion updates pace

1. User completes task
2. Toggles item to done
3. Completed count increases
4. Open count decreases
5. Pace percentage recalculates
6. Pace label updates

Outcome:
Progress feels visible and reinforcing.

## Flow 4 — Blocker escalation

1. User or manager identifies blocker
2. Blocker note is recorded or updated
3. Manager sees blocker in drill-down
4. Manager removes blocker or redirects task flow

Outcome:
Work does not stall silently.

## Flow 5 — Manager coaching review

1. Manager opens user view
2. Reads blocker note and standards
3. Reviews prompts such as:
   - what are your 3 highest-value actions before lunch?
   - if you finish early, what queue do you pull from?
   - what momentum did you create without being told?
4. Gives focused intervention

Outcome:
Coaching becomes specific, repeatable, and operational.
